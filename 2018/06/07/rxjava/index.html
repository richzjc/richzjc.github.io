学习rxjava 常用操作符

发表于 2018-06-07 | 分类于 android |
字数统计: 1,301

在android 开发中，充分利用rxjava编程会使得代码的可读性变得更强。
那在rxjava到底有哪些比较常用的操作符呢？接下来我主要对我自己在开发
中使用得不够熟练的操作符做一点总结。

just操作符

observable.just(1,2 3)
注意： just操作符最多只能传9个

fromArray

Integer[] a = {1, 2 ,3};
Observable.fromArray(a)
注意：传递的参数是一个数组，解决了just操作符的个数限制的缺点

fromCallback

Observable.fromCallable(new Callable<Integer>() {
            @Override
            public Integer call() throws Exception {
                return 1;
            }
        });
注意：参数为java.util.concurrent中的Callback对象

fromFuture(这个操作符不会用)

Observable.fromFuture(new Future<Integer>() {
            @Override
            public boolean cancel(boolean mayInterruptIfRunning) {
                return false;
            }

            @Override
            public boolean isCancelled() {
                return false;
            }

            @Override
            public boolean isDone() {
                return false;
            }

            @Override
            public Integer get() throws InterruptedException, ExecutionException {
                return null;
            }

            @Override
            public Integer get(long timeout, @NonNull TimeUnit unit) throws InterruptedException, ExecutionException, TimeoutException {
                return null;
            }
        });

fromIterable

ArrayList list;
Observable.fromIterable(list)
注意： 参数是实现Iterable接口的实例，通常情况下都是list

timer

Observable.timer(long delay, TimeUnit unit)
注意： 延迟delay后只发射一次事件，发射0

interval

Observable.interval()
看一下源码就知道每一个参数是什么意思了

intervalRange

Observable.intervalRange(long start, long count, long delay, long period, TimeUnit unit)
start: 开始
count: 发射的数量
delay: 发送第一次的延时
perios: 后续每发射一次的时间间隔
unit: 时间单位

range rangeLong

range(int start, int count)
rangeLong(long start, long count)
看参数就能明白什么意思

empty, nerver, error

Observable.empty()
Observable.error()
Observable.never()
研究一下这三个方法的使用场景

map

flatmap(无序)

concatMap(有序，对比flatmap的用法使用)

buffer

buffer(int count, int skip)

Observable.just(1,2,3,4)
.buffer(2, 1)

意思是：将发射的数据按照count 组合缓存起来统一发送

发射的数据是： （1，2）（2，3）（3，4）

groupBy

Observable.just(1,2,3,4,5,6)
                .groupBy(new Function<Integer, Integer>() {
                    @Override
                    public Integer apply(Integer integer) throws Exception {
                        return 1;
                    }
                })
                .map(new Function<GroupedObservable<Integer,Integer>, String>() {
                    @Override
                    public String apply(GroupedObservable<Integer, Integer> integerIntegerGroupedObservable) throws Exception {
                        return null;
                    }
                })

其实就是按照groupby方法里面的返回值进行分组

scan

Observable.just(1,2,3,4,5,6)
                .scan(new BiFunction<Integer, Integer, Integer>() {
                    @Override
                    public Integer apply(Integer s, Integer integer) throws Exception {
                        return null;
                    }
                })
一个聚合的操作符，类似于加法，

window

window(long count)

Observable.just(1,2,3,4)
.window(2)

类似于buffer(2,2)

concat concatArray

传的参数类型是Observable, concatArray解决了数量上的限制

merge mergeArray

原理和上面的是差不多的，只不过merge是并行发送的

concatArrayDelayError() & mergeArrayDelayError()

和concatArray， mergeArray用法差不多，唯一的区别在于，当抛异常后，onerror方法
会在所有事情发射完成后执行onerror

zip

会将多个被观察者合并，根据各个被观察者发送事件的顺序一个个结合起来，最终发送的事件数量会与源 Observable 中最少事件的数量一样。

Observable.zip(Observable.intervalRange(1, 5, 1, 1, TimeUnit.SECONDS)
    .map(new Function<Long, String>() {
        @Override
        public String apply(Long aLong) throws Exception {
            String s1 = "A" + aLong;
            Log.d(TAG, "===================A 发送的事件 " + s1);
            return s1;
        }}),
        Observable.intervalRange(1, 6, 1, 1, TimeUnit.SECONDS)
            .map(new Function<Long, String>() {
            @Override
            public String apply(Long aLong) throws Exception {
                String s2 = "B" + aLong;
                Log.d(TAG, "===================B 发送的事件 " + s2);
                return s2;
            }
        }),
        new BiFunction<String, String, String>() {
            @Override
            public String apply(String s, String s2) throws Exception {
                String res = s + s2;
                return res;
            }
        })
.subscribe(new Observer<String>() {
    @Override
    public void onSubscribe(Disposable d) {
        Log.d(TAG, "===================onSubscribe ");
    }

    @Override
    public void onNext(String s) {
        Log.d(TAG, "===================onNext " + s);
    }

    @Override
    public void onError(Throwable e) {
        Log.d(TAG, "===================onError ");
    }

    @Override
    public void onComplete() {
        Log.d(TAG, "===================onComplete ");
    }
});

reduce

与 scan() 操作符的作用也是将发送数据以一定逻辑聚合起来，
这两个的区别在于 scan() 每处理一次数据就会将事件发送给观察者，
而 reduce() 会将所有数据聚合在一起才会发送事件给观察者。

collect

用法：将数据收集起来
Observable.just(1, 2, 3, 4)
.collect(new Callable < ArrayList < Integer >> () {
    @Override
    public ArrayList < Integer > call() throws Exception {
        return new ArrayList < > ();
    }
},
new BiConsumer < ArrayList < Integer > , Integer > () {
    @Override
    public void accept(ArrayList < Integer > integers, Integer integer) throws Exception {
        integers.add(integer);
    }
})
.subscribe(new Consumer < ArrayList < Integer >> () {
    @Override
    public void accept(ArrayList < Integer > integers) throws Exception {
        Log.d(TAG, "===============accept " + integers);
    }
});

startWidth startWidthArray

在发送事件之前追加事件，startWith() 追加一个事件，startWithArray() 可以追加多个事件。追加的事件会先发出。
例：
Observable.just(5, 6, 7)
   .startWithArray(2, 3, 4)
   .startWith(1)
   .subscribe(new Consumer < Integer > () {
       @Override
       public void accept(Integer integer) throws Exception {
           Log.d(TAG, "================accept " + integer);
       }
   });
发射的数据是： 1，2 3，4，5，6，7

delay

延迟发射

doOnEach

Observable 每发送一件事件之前都会先回调这个方法。

doOnNext

doAfterNext

doOncomplete

doOnError

doOnDispose

doOnTerminate doOnAfterTerminate

doOnTerminate 是在 onError 或者 onComplete 发送之前回调，而 doAfterTerminate 则是 onError 或者 onComplete 发送之后回调

doFinally

在所有事件发送完毕之后回调该方法。
怎么用？
这里可能你会有个问题，那就是 doFinally() 和 doAfterTerminate()
到底有什么区别？区别就是在于取消订阅，如果取消订阅之后 doAfterTerminate() 就不会被回调，而 doFinally() 无论怎么样都会被回调，且都会在事件序列的最后。
